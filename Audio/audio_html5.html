Adding Sound

Audio in HTML5

HTML5 standardized the way we embed audio in websites with the Audio Tag. The audio tag is used to represent sound content in documents. (Notice that it represents the sound content, while the audio tag is capable of provided audio controls it is not representing them.)

There are several easy steps to embed sound using tags and to ensure that the sounds play across multiple browsers. This tutorial should take you 20 minutes. At the end, you'll have audio on your page.

controls

This optional attribute tells the browser to set up the native audio controls right where we have the audio tags. Without this attribute the represented sound would have no interface and it would be up to you to implement it.

There are several more attributes that the audio tag accepts. You can get a good handle on them by reading the Mozilla docs for the audio element.

Source Tags Inside Audio Tags

Let's dissect this:

The source tag is a special HTML5 tag that can be used inside the audio tag and the video tag. We are using two of the source tags attributes, the type attribute, which tells the browser the mime type or what type of file to expect, and the src attribute, which works just like the src attribute we used before on the audio tag.

I'm sure you noticed we are using two source tags. Defining your audio sources this way is beneficial because you can define more than one. We do this for browser support - not all browsers support all audio files in the same way. Some prefer and prioritize file types over others. Providing the browser with different options will help your sound run smooth and give you better support over all.

You can read more about the source tag on the Mozilla Developers Network

Being part of the new standard old browsers such as IE8 and IE7 do not support the audio tag and will simply ignore it, but will render any valid HTML inside of the tag correctly.
